\chapter{相关研究概述}[Related work]

\section{引言}[Introduction]

通过引入外部知识，RAG 有效缓解了大语言模型在知识过时与事实性错误方面的局限，使其在知识密集型任务中取得了显著进展。
然而，在复杂多跳问答等任务中，现有 RAG 方法仍面临上下文噪声较大、语义表达不充分以及推理能力受限等问题。
为提升 RAG 系统的推理能力，研究者逐渐引入结构化知识建模方法，通过构建知识图谱或语义图结构来组织检索信息，
从而增强多跳推理能力。这类方法能够显式建模实体及其关系，在一定程度上改善了传统 RAG 在复杂推理任务中的表现。
然而，由于知识图谱构建过程中存在信息抽取误差及结构不一致等问题，基于结构化知识的方法仍然容易引入噪声，并带来额外的计算开销。
与此同时，大语言模型自身的推理机制、文本表示方式以及信息抽取技术，也在检索增强生成过程中发挥着重要作用。
向量化表示与相似度检索为 RAG 提供了基础的语义匹配能力，但其往往忽略文本内部的结构信息；
命名实体识别与开放式信息抽取技术则为构建结构化知识提供了关键支撑，但也存在抽取质量不稳定的问题。
如何在结构化知识与原始文本语义之间实现有效融合，成为当前研究的核心挑战之一。

基于上述背景，本章从检索增强生成方法、结构化知识建模、大语言模型推理机制、文本表示与向量检索技术
，以及实体识别与开放式信息抽取等方面，对相关研究进行系统梳理与分析，为后续研究工作提供理论基础与技术支撑。

\section{检索增强生成技术}[RAG techniques]

\subsection{朴素检索增强生成}

朴素检索增强生成（Naive Retrieval-Augmented Generation）通常指这样一类方法：
首先针对用户查询从外部文本语料库中检索若干相关文档或文档块，然后将这些检索结果直接拼接到提示中，
最后交由大语言模型生成答案。与后续引入知识图谱、推理链或显式结构约束的方法不同，
该类方法主要依赖“语义检索 + 上下文拼接 + 生成回答”的基本范式完成知识增强，
因此具有实现简单、部署方便和通用性强等特点。
其核心思想可以概括为：利用外部非参数化知识库弥补大语言模型参数化知识的静态性与有限性，
从而提升模型在知识密集型任务中的事实性与时效性。
如\figref{fig: RAG 基本流程}所示，展示了朴素 RAG 的基本流程。

\begin{figure}[htbp]
	\centering
	\includegraphics[width=0.8\textwidth]{朴素RAG基本流程.png}
	\caption{朴素RAG基本流程}
	\label{fig: RAG 基本流程}
\end{figure}

从技术流程上看，朴素 RAG 一般包括知识切块与索引、查询编码与召回、上下文构建以及答案生成四个环节。
首先，将外部语料切分为段落级或块级文本单元，并建立稀疏或稠密索引；
然后，将用户问题编码为向量表示，与候选文本块进行相似度匹配，选取得分最高的 Top-$k$ 文档块；
接着，将这些文本块按照一定顺序拼接为上下文；最后，将问题与检索上下文一并输入生成模型，输出最终答案。
Lewis 等人提出的 RAG 明确给出了“参数化记忆 + 非参数化记忆”的统一框架，
而 In-Context RALM 和 REPLUG\cite{shi2024replug} 则进一步表明，即使不改动语言模型内部结构，
仅通过检索文本并直接拼接到输入中，也能够获得稳定的知识增强效果。

在检索模块上，朴素 RAG 早期多依赖 BM25\cite{bm25} 等稀疏检索方法，随后逐渐转向以 DPR 为代表的稠密检索框架。
DPR 使用双编码器分别对问题和文本块进行编码，并通过向量相似度实现高效检索，成为后续大量 RAG 系统的基础组件。
另一方面，REALM 则更早将外部文档检索机制引入预训练阶段，证明了语言模型可以在预训练、微调和推理阶段持续访问外部知识，
从而增强开放域问答能力。这些工作奠定了朴素 RAG 在“检索器 + 生成器”范式下的基本技术基础。

在生成模块上，朴素 RAG 既可以采用像 RAG 这样的“边检索边生成”方式，也可以采用将多个检索结果进行融合后再解码的方式。
Izacard\cite{izacard2021leveraging} 等人提出的 FiD（Fusion-in-Decoder）通过分别编码多个检索文本块，
并在解码阶段进行融合，显著提升了开放域问答性能，其框架如\figref{fig: FiD 框架图}所示。
后续 Atlas 则在此基础上进一步展示了检索增强模型在少样本知识密集型任务中的优势，
而 RETRO 则把检索更深地集成到语言模型架构中，通过从大规模文本库中检索邻近文本块增强自回归建模能力。
尽管这些方法在具体实现上有所不同，但本质上仍然遵循“从文本库中直接检索证据，再将文本证据输入生成模型”的朴素 RAG 思路。

\begin{figure}[htbp]
	\centering
	\includegraphics[width=0.98\textwidth]{fid框架图.png}
	\caption{FiD 框架\cite{izacard2021leveraging}}
	\label{fig: FiD 框架图}
\end{figure}

朴素 RAG 的优势在于实现路径清晰、工程成本较低，并且能够方便地接入现有大语言模型或黑盒 API。
特别是在无需改动模型内部参数或结构的场景下，In-Context RALM 与 REPLUG 这类方法展示了较强的实用价值。
与此同时，由于其知识源来自可更新的外部文本库，朴素 RAG 相比纯参数化大语言模型在知识时效性、可追溯性和可维护性方面具有明显优势。

然而，朴素 RAG 的局限性同样十分突出。
首先，该类方法通常基于问题与文本块之间的整体语义相似度进行检索，缺乏对实体关系和跨文档逻辑结构的显式建模，
因此在多跳推理场景下容易遗漏关键证据。其次，Top-$k$ 检索结果往往包含大量冗余和噪声，
将其直接拼接到输入上下文中会增加模型的阅读负担，并可能削弱最终答案的准确性与稳定性。
再次，为提升召回率而扩大检索范围时，输入 Token 数量会迅速增长，从而带来额外计算成本和上下文截断风险。
正是由于这些不足，后续研究开始尝试引入知识图谱、开放式信息抽取、图检索以及多粒度语义建模等机制，
以进一步提升检索增强生成在复杂推理任务中的能力。

\subsection{基于知识图谱的检索增强生成}

基于知识图谱的检索增强生成（Knowledge Graph-based Retrieval-Augmented Generation, KG-RAG）
是指在传统检索增强生成框架中，引入知识图谱作为知识组织与检索载体，
从而提升大语言模型在复杂问答与多跳推理任务中的知识利用能力。
与朴素 RAG 直接对文本块进行相似度检索不同，KG-RAG 通过显式建模实体、关系及其连接结构，
将原本分散在文档中的知识组织为可计算的图结构，使检索过程不仅能够返回“相关文本”，还能够提供“结构化证据”与“关系路径”。
这使得模型在面对跨文档、多实体和多步推理问题时，能够更容易捕捉关键线索并形成可解释的推理过程。
从知识来源角度来看，基于知识图谱的检索增强生成方法主要可以分为两类：一类是直接利用已有的公开知识图谱作为外部知识源，
另一类则是从原始文本中自动构建知识图谱以支持动态检索。

首先，基于公开知识图谱的方法通常依赖已有的大规模结构化知识库，如 Wikidata、DBpedia、UMLS 或 SPOKE 等。
这类方法通过实体链接将用户问题中的关键实体映射到知识图谱节点，并利用图结构进行邻居扩展或路径搜索，从而获取与问题相关的多跳知识。
如 Soman 等人提出的方法通过从生物医学知识图谱 SPOKE 中检索生物医学三元组构成上下文，
从而优化提示，使大模型生成的回复牢固地建立在完善的生物医学知识的基础上，其框架如\figref{fig: bio 框架图}所示。
KG-Rank 利用外部医学知识图谱 UMLS 替代传统 RAG 中的文本向量知识库，转而编码知识图谱中的三元组，
通过提取查询中的实体，检索知识图谱中相关的三元组，进行排序与重排序，形成最终的上下文。
这类方法的优势在于知识结构稳定、语义明确，但其效果依赖于图谱的覆盖范围与更新频率，在开放域场景中存在一定局限。

其次，随着大语言模型和开放式信息抽取技术的发展，越来越多的方法倾向于从文本中自动构建知识图谱，以提升方法的灵活性与适应性。
典型做法是利用 OpenIE 或大语言模型从文档中抽取实体关系三元组，构建面向当前语料的知识图结构。
从技术流程上看，其通常包括知识图谱构建、图索引与表示、图上检索与扩展以及结构化证据生成四个阶段。
首先，需要从已有知识库或原始文本中获取实体和关系信息，并构建知识图谱。
随后，通过实体嵌入、关系嵌入、图神经网络表示学习或图索引机制，为后续检索提供基础。
接着，在推理阶段，系统根据用户问题定位相关实体，并通过邻居扩展、路径搜索、图传播或社区聚合等方式，
在图中查找与问题相关的结构化证据。最后，再将这些图证据以路径、子图、摘要或图文融合上下文的形式输入大语言模型，
以支持最终答案生成。

在知识组织层面，知识图谱为 RAG 提供了显式的关系结构，这一点是其区别于朴素 RAG 的核心特征。
传统文本检索依赖整体语义相似度，容易忽略实体之间的组合关系，
而知识图谱则能够以三元组或子图形式保留“谁与谁存在何种关系”的结构信息。
例如，GraphRAG 通过先构建实体图，再利用社区划分与社区摘要实现由局部到全局的知识组织，
适合处理需要面向整个语料集合进行总结和全局分析的问题；
HippoRAG 则通过知识图谱和个性化 PageRank 模拟长期记忆检索机制，使得检索过程能够沿着图结构完成多跳扩展，
其框架如\figref{fig: HippoRAG 框架图}所示；
LightRAG 强调图结构上的低层实体检索与高层关系聚合，兼顾信息覆盖与检索效率。

在检索机制层面，KG-RAG 往往不再局限于一次性的 Top-$k$ 文本块召回，
而是将检索过程扩展为“实体定位—结构扩展—证据聚合”的多阶段过程。
典型做法包括：以问题中的实体为起点，在图中搜索相关邻居和关系路径；以图神经网络对局部子图进行编码，从而获取更加紧凑的检索表示；
或通过图上的排序与传播机制，估计不同节点、边和路径的重要性。
GNN-RAG 便是这一思路的代表，该方法利用图神经网络在知识图谱子图上执行稠密推理，先找到候选答案，
其框架如\figref{fig: GNN-RAG架构图}所示，
再抽取连接问题实体与候选答案的最短路径，并将其语言化后输入大语言模型，从而实现“图推理 + 文本生成”的协同。
类似地，GFM-RAG 则尝试利用图基础模型增强图表示与图检索能力，以改善复杂图结构上的泛化效果。

在上下文构建层面，基于知识图谱的方法通常会将结构化知识与文本证据结合使用，而不是仅返回图中的节点或边。
一方面，知识图谱中的路径、子图和社区摘要能够为问题提供更清晰的逻辑框架；
另一方面，原始文本仍然承载着更丰富的语义细节。为此，许多 KG-RAG 方法采用“结构化检索 + 文本补充”的方式构建提示。
例如，GraphRAG 使用社区摘要作为全局上下文，HippoRAG 将图上的传播结果与文本证据结合，
LightRAG 则通过图结构引导文本检索，增强上下文的可解释性与完整性。
可以看出，知识图谱在这类方法中并非替代文本，而是作为组织和筛选文本的重要中介。

基于知识图谱的检索增强生成具有多方面优势。
首先，它能够显式建模实体关系，从而更适合多跳问答、复杂推理和跨文档证据聚合等任务。
其次，图结构本身具有较强的可解释性，便于分析模型回答所依赖的路径和证据来源。
再次，通过图上的路径搜索、传播和聚合，能够在一定程度上缓解朴素 RAG 中“仅按表面语义相似度检索”导致的证据割裂问题。
但与此同时，KG-RAG 也存在明显局限。其一，知识图谱的质量直接决定检索与推理效果，
如果图中存在抽取错误、实体歧义或结构断裂，则可能将错误信息传播到后续回答过程中。
其二，图谱构建、图表示学习与图检索通常会带来额外的时间与存储成本，尤其是在大规模语料场景下，这一问题更为突出。
其三，尽管图结构有利于表达关系，但结构化表示本身较为稀疏，往往不足以替代原始文本中的丰富语义，
因此如何高效融合“图结构”和“文本上下文”，仍然是该方向中的关键问题。

\section{文本表示与向量检索技术}

文本表示与向量检索是检索增强生成系统的基础环节，其核心目标是将自然语言文本映射到可计算的表示空间，并通过相似度计算实现高效检索。
在传统信息检索中，文本通常采用基于词频统计的稀疏表示方法，如 BM25 等模型。
这类方法具有实现简单、可解释性强的优点，在关键词匹配明确的场景中依然表现稳定；
但由于其主要依赖词项重叠，难以刻画深层语义关联，因此在语义表达丰富、措辞多样或需要隐式推理的任务中存在明显局限。

随着预训练语言模型的发展，文本表示逐渐从稀疏词袋表示转向稠密向量表示。
早期的 BERT\cite{bert} 类模型虽然具备较强的上下文建模能力，但直接用于语义相似度计算时效率较低。
为此，Reimers\cite{reimers2019sentence} 等人提出了 Sentence-BERT（SBERT），通过孪生网络结构学习句子级向量表示，
使文本能够直接在向量空间中进行相似度比较，大幅提升了语义检索与聚类等任务的效率。
此后，MPNet\cite{mpnet} 进一步结合掩码语言建模与排列语言建模的优点，提高了预训练表示的语义质量，
也成为后续多种句向量模型的重要基础，其框架如\figref{fig: MPNets框架图}所示。

在检索任务中，稠密表示通常通过双编码器（dual-encoder）框架实现。其基本思想是分别对查询和候选文本进行独立编码，
再通过向量内积或余弦相似度计算两者相关性。
DPR 是这一方向的代表性工作，其表明仅依赖稠密向量检索便能够在开放域问答任务中显著优于传统 BM25 基线。
近年来，随着通用文本嵌入模型的发展，E5\cite{e5} 等模型通过弱监督对比学习进一步提升了向量表示的泛化能力，
在检索、聚类和分类等多种任务上表现出较强的迁移性能。这使得文本嵌入模型逐渐成为现代检索增强生成系统中的核心组件。

在相似度计算方面，向量检索通常采用内积（inner product）或余弦相似度（cosine similarity）作为度量标准。
设查询向量为 $\mathbf{q} \in \mathbb{R}^d$，候选文本向量为 $\mathbf{d} \in \mathbb{R}^d$，则内积相似度定义为：

\begin{equation}
\text{sim}_{\text{dot}}(\mathbf{q}, \mathbf{d}) = \mathbf{q}^\top \mathbf{d} = \sum_{i=1}^{d} q_i d_i
\end{equation}

余弦相似度则在内积基础上对向量进行归一化处理，其定义为：

\begin{equation}
\text{sim}_{\cos}(\mathbf{q}, \mathbf{d}) = \frac{\mathbf{q}^\top \mathbf{d}}{\|\mathbf{q}\| \, \|\mathbf{d}\|}
\end{equation}

其中，$\|\mathbf{q}\|$ 和 $\|\mathbf{d}\|$ 分别表示向量的 $\ell_2$ 范数。
在实际应用中，当向量经过归一化处理后，内积相似度与余弦相似度在排序效果上通常等价，因此常使用内积来实现更高效的计算。

由于大规模语料库中的候选文本数量通常非常庞大，若直接对所有向量逐一比对，将带来较高的时间开销。
因此，研究者通常借助近似最近邻（Approximate Nearest Neighbor, ANN）索引技术提升检索效率。
FAISS\cite{faiss} 是这一方向中应用最广泛的工具之一，它支持多种高维向量索引与加速检索策略，
能够在保证较高召回率的同时显著降低大规模相似度搜索的计算成本。

尽管基于文本嵌入的向量检索方法在语义匹配方面具有显著优势，但其局限性同样值得关注。
首先，单一向量表示通常会将整段文本压缩为统一语义表示，难以显式反映文本内部不同句子或不同语义单元的重要性差异。
其次，基于整体语义相似度的检索方式虽然能够提高召回能力，但也可能引入与问题仅在表面语义上相关、而在推理层面并不关键的文本片段。
尤其是在多跳问答场景中，如果缺乏对实体、短语或局部语义结构的进一步建模，仅依赖整体向量表示容易导致关键证据被淹没于冗余上下文中。
正因如此，如何在向量检索的基础上进一步引入更细粒度的语义表示与结构化建模，成为本文后续研究的重要出发点。

\section{命名实体识别与开放式信息抽取技术}

命名实体识别（Named Entity Recognition, NER）
与开放式信息抽取是构建结构化知识表示的重要基础技术。
前者主要负责从自然语言文本中识别具有明确语义指称的实体片段及其类别，如人名、地名、机构名、时间表达等；
后者则进一步从句子中抽取开放域关系事实，通常表示为三元组或关系元组形式。
二者共同构成了从非结构化文本中获取结构化知识的关键途径，并被广泛应用于知识图谱构建、问答系统、信息检索和检索增强生成等任务。

\subsection{命名实体识别技术}

命名实体识别的目标是：给定一个输入文本序列
\begin{equation}
X = \{x_1, x_2, \dots, x_n\},
\end{equation}
预测与之对应的标签序列
\begin{equation}
Y = \{y_1, y_2, \dots, y_n\},
\end{equation}
其中 $y_i$ 表示第 $i$ 个词或子词对应的实体标签。实际系统中通常采用 BIO 或 BIOES 等标注方案，
将实体边界识别与实体类别判断统一为序列标注问题。其目标可以形式化表示为
\begin{equation}
Y^{*} = \arg\max_{Y} P(Y \mid X).
\end{equation}

如\figref{fig: NER框架}所示，展示了命名实体识别任务的基本流程。
早期命名实体识别方法主要依赖人工设计特征和统计学习模型，如条件随机场（Conditional Random Field, CRF）等。
这类方法需要结合词性、词形、上下文窗口以及词典特征进行建模，具有较强可解释性，但对特征工程依赖较强，迁移能力有限。
随着深度学习的发展，基于双向长短期记忆网络（BiLSTM）与 CRF 的神经序列标注模型逐渐成为主流\cite{lample2016neural}。
Lample 等人提出的 BiLSTM-CRF 架构表明，在无需复杂人工特征的前提下，
仅利用字符级表示与词向量即可在多个 NER 基准上取得优异效果。

	\centering
	\includegraphics[width=0.90\textwidth]{命名实体识别.png}
	\caption{命名实体识别基本流程}
	\label{fig: NER框架}
\end{figure}

在此基础上，预训练语言模型进一步推动了 NER 技术的发展。
BERT 通过双向 Transformer 预训练获得上下文相关表示，使得 NER 能够充分利用更丰富的上下文语义信息。
基于 BERT 的命名实体识别通常将输入文本编码为上下文向量，再在输出层进行逐位置分类，或结合 CRF 层建模标签之间的依赖关系。
设 BERT 编码器输出的上下文表示为
\begin{equation}
\mathbf{H} = \{\mathbf{h}_1, \mathbf{h}_2, \dots, \mathbf{h}_n\},
\end{equation}
则每个位置的标签概率可表示为
\begin{equation}
P(y_i \mid X) = \text{Softmax}(W \mathbf{h}_i + b),
\end{equation}
其中 $W$ 和 $b$ 为分类层参数。BERT 及其变体在命名实体识别任务中显著提升了性能，尤其适用于上下文依赖较强的复杂实体识别场景。

从本文研究内容来看，命名实体识别的作用主要体现在两个方面：一方面，它是构建图结构或语义索引的起点，
能够从原始文本中定位关键实体节点；另一方面，在后续多粒度语义建模中，实体通常也是最基本、最稳定的语义单元。
需要指出的是，NER 虽然能够较好地识别显式实体，但其识别对象通常局限于具有明确边界和类型的命名实体，
对于“政策变化”“神经网络结构”等不一定属于标准命名实体但具有重要语义价值的短语级概念，覆盖能力相对不足。
这也是后续需要引入短语抽取等更细粒度语义建模方式的重要原因。

\subsection{开放式信息抽取技术}

开放式信息抽取的目标是从任意领域文本中自动抽取关系事实，而不预先限定关系模式。与传统关系抽取需要预定义关系类别不同，
OpenIE 更强调开放域、可扩展和领域无关的抽取能力。给定一句自然语言文本 $s$，OpenIE 系统通常输出若干关系事实：
\begin{equation}
\mathcal{T}(s) = \{(h_1, r_1, t_1), (h_2, r_2, t_2), \dots, (h_m, r_m, t_m)\},
\end{equation}
其中 $h_i$ 表示头实体或主语成分，$r_i$ 表示关系短语，$t_i$ 表示尾实体或宾语成分。对于整个文档集合 $\mathcal{D} = \{d_1, d_2, \dots, d_k\}$，可进一步得到三元组总集合
\begin{equation}
\mathcal{T}_{\mathcal{D}} = \bigcup_{i=1}^{k} \mathcal{T}(d_i).
\end{equation}

如\figref{fig: OpenIE示意图}所示，展示了开放式信息抽取的基本流程。
OpenIE 的代表性工作可以追溯到 Etzioni\cite{openie} 等人提出的 TextRunner 框架，其核心思想是从海量文本中自动抽取开放域关系，
而不依赖预定义关系模板。这一工作被普遍视为开放式信息抽取方向的开端。
随后，Schmitz\cite{schmitz2012open} 等人提出 OLLIE，在依存句法分析基础上进一步提升了抽取质量，能够处理更加复杂的语义结构，
如归因关系和条件关系，从而显著增强了开放式抽取的表达能力。

	\centering
	\includegraphics[width=0.90\textwidth]{OpenIE示意图.png}
	\caption{OpenIE基本流程}
	\label{fig: OpenIE示意图}
\end{figure}

从技术实现上看，OpenIE 方法大致可以分为三类。
第一类是基于规则与模式匹配的方法，通过句法模板、依存路径和启发式规则抽取关系；
第二类是基于监督或弱监督学习的方法，通过自动构造训练数据学习抽取器；
第三类是近年来兴起的基于大语言模型的方法，即通过指令提示或上下文学习，直接让大语言模型从句子或段落中生成关系三元组。
相较于传统 OpenIE，基于 LLM 的方法在复杂句法和跨句语义理解方面具有更强能力，也更容易适配不同领域语料，
因此在近年来的知识图谱构建与图增强 RAG 研究中被广泛采用。

在检索增强生成场景下，开放式信息抽取的主要作用是：将原始文本中的隐式知识关系转化为显式的结构化表示。
这样一来，系统可以围绕实体、关系和路径进行检索、扩展和重排序，而不再仅依赖文本块级别的整体语义相似度。
尤其是在多跳问答任务中，OpenIE 提供的三元组能够帮助系统显式组织跨文档证据，从而增强推理能力和可解释性。

从技术流程上看，命名实体识别与开放式信息抽取具有明显的层次关系。
命名实体识别更关注“识别文本中的关键实体单元”，而开放式信息抽取则进一步关注“识别实体之间的关系及其事实表达”。
前者为后者提供候选语义节点，后者则为前者补充结构化关系信息。可以将两者的关系概括为：
\begin{equation}
\text{Text} \xrightarrow{\text{NER}} \text{Entities} \xrightarrow{\text{OpenIE}} \text{Triples / Graph}.
\end{equation}

然而，随着大语言模型的发展，传统的“先实体识别、再关系抽取”的流水线式处理方式逐渐被统一建模方式所替代。
在基于大语言模型的开放式信息抽取中，模型可以直接从原始文本中同时识别实体及其关系，
即在无需显式命名实体识别模块的情况下，直接生成结构化三元组：
\begin{equation}
\text{Text} \xrightarrow{\text{LLM-based OpenIE}} \text{Triples}.
\end{equation}

该方式利用大语言模型强大的上下文理解能力，将实体识别与关系抽取过程进行隐式融合，
能够更好地处理复杂句法结构与跨句语义关系，因此在近年来的知识图谱构建与图增强检索方法中得到广泛应用。
然而，该类方法也存在一定局限，例如生成结果的稳定性较弱、结构约束不足以及容易引入噪声等问题。
因此，在实际系统设计中，仍需根据任务需求在“显式分阶段抽取”和“统一生成式抽取”之间进行权衡。

总体而言，命名实体识别与开放式信息抽取是从非结构化文本中构建结构化知识的两项基础技术。
命名实体识别适合提取稳定、明确的核心语义节点，而开放式信息抽取则能够进一步揭示实体之间的关系结构，
为多跳推理和图增强检索提供支持。与此同时，这两类技术也各自存在局限：NER 对短语级、概念级语义覆盖不足，
OpenIE 则容易受到抽取噪声和关系表达不稳定的影响。如何根据具体任务需求，
在实体、短语、三元组与原始文本之间建立更高效的协同关系，是后续方法设计中的关键问题。

随后，介绍了文本表示与向量检索技术，为高效语义匹配与大规模检索提供基础支撑；
最后，阐述了命名实体识别与开放式信息抽取技术在结构化知识构建中的作用。
